### 2. URL 格式分析

**V41.133 生成的 URL 格式**:
```
/football/{country}/{league-slug}/{home-slug}-{away-slug}-{match_id}/
```

**示例**:
- `/football/italy/serie-a/sassuolo-napoli-3657143/`
- `/football/germany/bundesliga/union-berlin-borussia-dortmund-3399269/`

**实际 OddsPortal URL 格式**（来自现有 L3 数据）:
```
/football/{country}/{league-slug}/{home-slug}-{away-slug}-{hash}/
```

**示例**:
- `/football/england/premier-league/everton-nottingham-forest-ID0097YDEK/`
- `/football/england/premier-league/liverpool-bournemouth-ID0020QENP/`

**关键差异**:
- V41.133 使用 `match_id`（数字）作为后缀
- 实际 OddsPortal 使用独特的 `hash`（字母数字组合）如 `ID0097YDEK`

**影响**:
- URL 格式不正确，但由于代理封禁，无法验证修正后的效果
- 即使格式正确，仍然需要真实的 hash 后缀

    """V41.133: 模拟 OddsPortal 数据（基于 FotMob 数据）"""
    odds_id = fotmob_match["match_id"]

    def slugify(name: str) -> str:
        return name.lower().replace(" ", "-")

    home_team_slug = slugify(fotmob_match["home_team"])
    away_team_slug = slugify(fotmob_match["away_team"])

    return {
        ...
        "url_path": url_path  # V41.133: 新增
    }
