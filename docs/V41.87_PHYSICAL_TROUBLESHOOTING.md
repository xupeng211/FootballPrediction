## 🔍 根本原因分析

### 为什么出现 ClientConnectorError？

**ClientConnectorError** 的含义：
- WSL2 客户端尝试连接到 `172.25.16.1:XXXX`
- 但宿主机上的**没有人监听这些端口**
- 操作系统返回 `ECONNREFUSED` (连接被拒绝)

### 步骤 3: 修复方案 (3 个选择)

#### 方案 A: 使用 v2rayN (推荐，支持多端口)

**v2rayN 支持多端口配置**:

1. **下载 v2rayN**: https://github.com/2dust/v2rayN/releases

2. **配置多个入站协议**:
   - 打开 v2rayN
   - 参数设置 → 核心设置 → 找到 "inbounds" 配置
   - 添加多个入站协议，每个使用不同端口

3. **示例配置** (config.json):
```json
{
  "inbounds": [
    {
      "port": 7891,
      "protocol": "http",
      "listen": "0.0.0.0"
    },
    {
      "port": 7892,
      "protocol": "http",
      "listen": "0.0.0.0"
    },
    {
      "port": 7893,
      "protocol": "socks",
      "listen": "0.0.0.0"
    },
    // ... 继续添加到 7909
  ]
}
```

4. **重启 v2rayN**

**时间成本**: ~20 分钟

---

#### 方案 B: 使用 Clash 代理链 + 端口转发

**原理**: 使用 5 个可用端口，通过端口转发实现 19 个逻辑端口

**步骤**:
1. 保留 5 个可用端口 (7891-7895)
2. 使用 Windows 的 `netsh` 命令添加端口转发

```powershell
# 添加端口转发 (7896 → 7891, 7897 → 7892, ...)
netsh interface portproxy add v4tov4 listenport=7896 listenaddress=0.0.0.0 connectport=7891 connectaddress=127.0.0.1
netsh interface portproxy add v4tov4 listenport=7897 listenaddress=0.0.0.0 connectport=7892 connectaddress=127.0.0.1
netsh interface portproxy add v4tov4 listenport=7898 listenaddress=0.0.0.0 connectport=7893 connectaddress=127.0.0.1
# ... 继续添加

# 查看所有转发规则
netsh interface portproxy show all

# 删除转发规则（如需）
netsh interface portproxy delete v4tov4 listenport=7896 listenaddress=0.0.0.0
```

**缺点**: 端口转发不增加 IP 多样性（多个逻辑端口共享同一物理 IP）

**时间成本**: ~30 分钟
