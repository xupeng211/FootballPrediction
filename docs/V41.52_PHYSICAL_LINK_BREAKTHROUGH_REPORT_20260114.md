## 1. 物理环境诊断 (P0)

### 1.1 问题定位

**症状**:
- `localhost:5432` 连接成功
- 但数据库为空（0 张表，matches 表不存在）

**诊断命令**:
```bash
# 检查端口占用
ss -tulpn | grep 5432

# 输出: tcp LISTEN 0 244 127.0.0.1:5432
# 说明有服务在监听，但不是 Docker 的 PostgreSQL
```

**根本原因**:
主机上运行着另一个 PostgreSQL 实例，占用了 5432 端口，导致连接被路由到错误的数据库。

### 1.2 Docker 容器状态

```bash
docker-compose ps db
# football_prediction_db   postgres:15-alpine   Up 12 hours (healthy)
# PORTS: 0.0.0.0:5432->5432/tcp
```

**网络配置**:
- 网络: `football_prediction_network`
- 容器 IP: `172.20.0.3`
- 端口映射: `0.0.0.0:5432->5432/tcp`

### 1.3 证据对比

| 连接方式 | 表数量 | matches 表 | 数据库 |
|---------|-------|-----------|--------|
| localhost:5432 | **0** | ❌ 不存在 | 空数据库 |
| docker-compose exec | **14** | ✅ 9,069 场 | 真实数据库 |
