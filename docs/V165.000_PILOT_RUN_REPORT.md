## 二、根本原因分析

### 2.1 数据流发现

通过深度诊断，发现了 **OddsPortal 数据处理的真相**：

```
┌─────────────────────────────────────────────────────────────┐
│  1. 页面加载                                                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  2. XHR: /match-event/1-1-jVjHFI03-1-2-*.dat                │
│     • 响应大小: 18536 bytes                                  │
│     • 格式: Base64 编码字符串                                │
│     • 示例: L1UwRUtTS01tZUNhNVJRNkVU...                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  3. atob() 解码 (25次调用)                                   │
│     • Base64 → 压缩数据                                      │
│     • 输出: /U0EKSKMmeCa5RQ6ETuWfoNP2QZGPS...                │
│     • 格式: JXG 压缩格式                                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  4. ❌ JXG.decompress 未被调用 (0次)                        │
│     • 预期: JXG.decompress() → JSON                         │
│     • 实际: 未发生解压调用                                   │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  5. 结果: 数据未被捕获                                      │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键发现

**诊断结果**：
```
atob() 调用:     25 次  ✅ (Base64 解码正常)
JXG 库可用:      true  ✅ (JXG 已加载)
JXG.decompress:  0 次  ❌ (未被调用!)
JSON.parse:      500+ 次
```

**核心问题**：
1. `/match-event/` 返回的确实是 JXG 压缩数据（Base64 编码）
2. 页面调用 `atob()` 解码 Base64
3. **但 `JXG.decompress` 从未被调用！**
4. 这说明页面使用了**不同的解压方法**或**数据格式已变更**

### 2.3 可能原因

| 原因 | 可能性 | 说明 |
|------|--------|------|
| 页面使用自定义解压函数 | 🔴 高 | 可能封装了 JXG.decompress |
| 数据格式已变更 | 🟡 中 | 可能不再使用 JXG 压缩 |
| 解压在 Worker 中进行 | 🟡 中 | 主线程无法拦截 |
| 异步解压流程 | 🟢 低 | 使用 Promise/async 包装 |

```javascript
// 在 SignalRadar.js 中添加 XHR 响应钩子
page.on('response', async (response) => {
    const url = response.url();
    if (url.includes('/match-event/') && url.includes('.dat')) {
        const text = await response.text();

        // 1. Base64 解码
        const decoded = atob(text);

        // 2. 手动调用 JXG.decompress
        const decompressed = await page.evaluate((data) => {
            if (window.JXG && window.JXG.decompress) {
                return window.JXG.decompress(data);
            }
            return null;
        }, decoded);

        // 3. 解析并存储
        if (decompressed) {
            const json = JSON.parse(decompressed);
            // ... 处理数据
        }
    }
});
```
