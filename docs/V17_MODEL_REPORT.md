1. **V16.0 的 96.25% 准确率是"虚高"**：使用了赛后统计数据（xG、射门、控球率等），这些数据在比赛前不可用

3. **特征维度大幅降低**：从 223 维减少到 16 维，但预测能力是真实的
4. **Draw 类别预测困难**：F1 Score 为 0.00，需要在 V17.1 中改进

## D. 模型参数配置

### XGBoost 超参数

| 参数 | 值 | 说明 |
|------|------|------|
| `n_estimators` | 200 | 树的数量 |
| `max_depth` | 4 | 树的最大深度（防止过拟合） |
| `learning_rate` | 0.05 | 学习率（保守） |
| `min_child_weight` | 3 | 最小子节点权重 |
| `reg_alpha` | 0.5 | L1 正则化 |
| `reg_lambda` | 1.0 | L2 正则化 |
| `subsample` | 0.8 | 行采样比例 |
| `colsample_bytree` | 0.8 | 列采样比例 |
| `objective` | multi:softprob | 多类别概率输出 |
| `eval_metric` | mlogloss | 多类别对数损失 |

### 正则化策略

```
正则化强度: 中等
- L1 (reg_alpha=0.5): 特征选择，鼓励稀疏解
- L2 (reg_lambda=1.0): 防止过拟合，平滑权重

采样策略:
- 行采样 (subsample=0.8): 每棵树使用 80% 数据
- 列采样 (colsample_bytree=0.8): 每棵树使用 80% 特征
```

```python
# 预测接口示例
import joblib
import pandas as pd

def predict_match(home_team: str, away_team: str, match_date: str):
    """赛前预测接口"""
    # 1. 加载模型
    model = joblib.load('src/production_models/v17.0_rolling_model.pkl')

    # 2. 获取滚动特征
    rolling_features = get_rolling_features(home_team, away_team, match_date)

    # 3. 预测
    prediction = model.predict([rolling_features])[0]
    probability = model.predict_proba([rolling_features])[0]

    # 4. 返回结果
    return {
        'prediction': ['Away', 'Draw', 'Home'][prediction],
        'probabilities': {
            'Away': probability[0],
            'Draw': probability[1],
            'Home': probability[2]
        },
        'confidence': probability.max()
    }
```
