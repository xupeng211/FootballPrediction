## 🔒 任务 1: 数据库硬红线约束 (P0)

### 约束迁移脚本
**文件**: `scripts/sql/v41_32_add_unique_constraint.sql`

### 实施的约束

#### 1. 部分唯一索引 (Partial Unique Index)
```sql
CREATE UNIQUE INDEX idx_matches_mapping_oddsportal_hash_unique
ON matches_mapping (oddsportal_hash)
WHERE oddsportal_hash IS NOT NULL
  AND LENGTH(oddsportal_hash) = 8;
```

**特性**:
- 只对非 NULL 且长度为 8 的哈希进行唯一性约束
- 允许多个 NULL 哈希记录共存
- B-tree 索引，插入时间复杂度 O(log n)

#### 2. 触发器自动检测
```sql
CREATE OR REPLACE FUNCTION check_unique_hash_before_insert()
RETURNS TRIGGER AS $$
BEGIN
    IF EXISTS (
        SELECT 1
        FROM matches_mapping
        WHERE oddsportal_hash = NEW.oddsportal_hash
          AND oddsportal_hash IS NOT NULL
          AND LENGTH(oddsportal_hash) = 8
          AND fotmob_id != NEW.fotmob_id
    ) THEN
        RAISE EXCEPTION 'Duplicate hash "%" detected.', NEW.oddsportal_hash;
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

**特性**:
- 在 INSERT 和 UPDATE 时触发
- 应用层级重复检测
- 提供清晰的错误信息

### 算法设计

#### HashRecoveryEngine 类
```python
class HashRecoveryEngine:
    def calculate_match_score(self, target: Dict, candidate: Dict) -> float:
        score = 0.0

        # 1. 队名相似度（权重 50%）- Levenshtein 距离
        team_similarity = self.calculate_team_similarity(target, candidate)
        score += team_similarity * 0.5

        # 2. 同联赛加分（权重 20%）
        if self.is_same_league(target, candidate):
            score += 20.0

        # 3. 日期邻近加分（权重 15%）
        if self.is_date_proximity(target, candidate):
            score += 15.0

        # 4. 主客场互换（权重 15%）- 已移除
        # V41.32 修复：主客场不应共享哈希

        return score
```

## 🛠️ 技术创新点

### 1. 部分唯一索引 (Partial Unique Index)

**创新**: PostgreSQL 部分索引实现条件唯一性

```sql
CREATE UNIQUE INDEX idx_matches_mapping_oddsportal_hash_unique
ON matches_mapping (oddsportal_hash)
WHERE oddsportal_hash IS NOT NULL
  AND LENGTH(oddsportal_hash) = 8;
```

**优势**:
- 允许多个 NULL 哈希记录共存
- 只约束有效的 8 位哈希
- 性能优于普通 UNIQUE 约束

### 2. 触发器自动检测

**创新**: 应用层重复检测 + 系统级约束双重保护

```sql
CREATE TRIGGER trg_check_unique_hash_before_insert
    BEFORE INSERT OR UPDATE OF oddsportal_hash
    ON matches_mapping
    FOR EACH ROW
    EXECUTE FUNCTION check_unique_hash_before_insert();
```

**优势**:
- 提供清晰的错误信息
- 应用层拦截重复数据
- 配合唯一索引实现双重保护

### 3. 版本号仲裁算法

**创新**: 提取 Mapping Method 版本号进行智能仲裁

```python
def extract_version(self, mapping_method: str) -> float:
    match = re.search(r'v?(\d+)\.?(\d+)?(?:\.?(\d+))?', mapping_method)
    if match:
        major = int(match.group(1))
        minor = int(match.group(2)) if match.group(2) else 0
        patch = int(match.group(3)) if match.group(3) else 0
        return major + minor / 100.0 + patch / 10000.0
    return 0.0
```

**版本号排序**:
- collision_v40_9 (40.9) > collision_v40_7 (40.7)
- hybrid_v40_3_10 (40.310) > season_sweep_v4 (4.0)
