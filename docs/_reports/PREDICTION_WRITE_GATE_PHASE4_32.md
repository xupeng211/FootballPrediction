## Prediction Entry Risk Audit

Observed package scripts:

- `npm run predict` runs `scripts/ops/predict_pipeline.py`
- `npm run predict:dry` runs `scripts/ops/predict_pipeline.py --dry-run`
- `npm run predict:json` runs `scripts/ops/predict_pipeline.py --format json`
- `npm run train`, `npm run train:fast`, and `npm run train:deep` run `scripts/ops/train_model.py`

Risk summary:

- Prediction scripts enter model pipeline code and are not used as Phase 4.32-safe previews.
- `predict:dry` is not treated as a safe no-op gate because it can still enter model loading / inference flow.
- Model artifacts are intentionally not loaded in this phase.
- Any path that writes `predictions` remains blocked unless separately authorized.
- Training remains blocked because it requires a multi-match finished historical dataset and artifact policy.


- Does not load a model artifact.

- `--commit` exits non-zero immediately.
