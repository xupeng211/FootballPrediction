## 🔄 工作流程

### 修复生命周期
1. **缺陷发现** → 自动扫描或手动报告
2. **优先级评估** → AI分析影响和紧急程度
3. **修复策略** → 选择AI修复、人工修复或混合模式
4. **执行修复** → 应用修复方案
5. **验证测试** → 确认修复效果
6. **质量检查** → 确保不引入新问题
7. **部署上线** → 合并到目标分支
8. **效果跟踪** → 长期监控修复效果

### 触发条件
- **定时扫描**: 每日自动全量扫描
- **PR触发**: 代码提交时增量扫描
- **测试失败**: 自动分析失败原因
- **手动触发**: 开发者主动启动
- **监控告警**: 系统异常自动触发

## 📝 使用说明

### 开发者操作
```bash
# 手动触发缺陷分析
make bugfix-analysis

# 查看修复建议
cat docs/_reports/BUGFIX_TODO.md

# 应用AI修复
python scripts/apply_ai_fix.py <task_id>

# 验证修复效果
make verify-fix
```

### AI助手操作
1. 定期检查此文件获取待修复任务
2. 按优先级执行修复任务
3. 更新任务状态和修复结果
4. 记录修复时间和效果数据

### 配置管理
- 修复策略配置: `config/bugfix_config.yaml`
- 质量门禁配置: `config/quality_gates.yaml`
- AI修复规则: `config/ai_repair_rules.yaml`
