## 🔍 架构洞察

### OddsPortal 数据流架构

```
┌─────────────────────────────────────────────────────────────┐
│                    OddsPortal 数据流                       │
└─────────────────────────────────────────────────────────────┘

页面加载阶段:
├── 请求 /match-event/*.dat (Base64 + 加密的二进制数据)
├── 请求 /feed/postmatch-score/*.dat (比分数据)
└── 请求 /ajax-* (用户相关数据)

数据处理阶段:
├── JavaScript 解密 .dat 文件
├── 解析数据到内部状态 (非 Vue state，可能是闭包)
└── DOM 初始化 (不含赔率数据)

用户交互阶段 (悬停):
├── 触发 mouseover 事件
├── JavaScript 从内部状态读取赔率数据
├── 动态生成 Tooltip DOM 元素
└── 插入 "Opening odds:..." 文本到 DOM
```

### 数据保护机制

| 层级 | 保护措施 | 破解难度 |
|------|---------|---------|
| 传输层 | HTTPS | ⭐ |
| 编码层 | Base64 | ⭐ |
| 加密层 | 未知 (可能是 AES/自定义) | ⭐⭐⭐⭐ |
| 混淆层 | JS 代码混淆 | ⭐⭐⭐ |
| 动态生成 | 悬停时才生成 DOM | ⭐⭐ |

**总破解难度**: ⭐⭐⭐⭐⭐ (企业级保护)

## 🛡️ 技术细节

### .dat 文件分析

**文件命名模式**:
```
match-event/{version}-{match_id}-{unknown1}-{unknown2}-{hash}.dat?geo={country}&
```

**示例 URL**:
```
/match-event/1-1-CEnSxfJ0-1-2-2a02c18434ec4b8ac9f92b25e7e0f847.dat?geo=SG&
```

**参数推测**:
- `1`: 可能是版本号或格式版本
- `CEnSxfJ0`: match_id
- `1-2`: 可能是 sport_id 或 format_id
- `2a02c18434ec4b8ac9f92b25e7e0f847`: 哈希或签名
- `.dat`: 数据文件扩展名
- `geo=SG`: 地理位置 (新加坡)
