## 🔧 V41.106 核心修复

### 修复 1: NaN → None 转换 (PostgreSQL JSONB 兼容性)

**问题**: PostgreSQL JSONB 不接受 `NaN` 值

**修复** (`src/api/collectors/fotmob_core.py:895-899`):
```python
# V41.106: 转换 NaN 为 None (PostgreSQL JSONB 兼容性修复)
# PostgreSQL JSONB 不接受 NaN，需要转换为 null
for key, value in features.items():
    if isinstance(value, float) and np.isnan(value):
        features[key] = None
```

**影响**: 29 个 NaN 特征现在存储为 `null`，可以正常插入数据库

### 修复 2: SchemaManager 初始化

**修复** (`src/database/schema_manager.py:87-96`):
```python
-- V41.106: 添加 technical_features 字段 (152维技术特征)
IF NOT EXISTS (
    SELECT 1 FROM information_schema.columns
    WHERE table_name = 'matches' AND column_name = 'technical_features'
) THEN
    ALTER TABLE matches ADD COLUMN technical_features JSONB DEFAULT '{}'::jsonb;
    RAISE NOTICE '✅ technical_features 字段已添加';
ELSE
    RAISE NOTICE 'ℹ️ technical_features 字段已存在';
END IF;
```

**影响**: 所有未来数据库初始化都会包含 `technical_features` 字段

## 🎯 使用指南

### 标准采集方式

```python
from src.api.collectors.fotmob_core import FotMobCoreCollector

# 创建采集器实例
collector = FotMobCoreCollector()

# 采集比赛数据（标准入口）
match_data = collector.fetch_match_details(match_id=3428850)

# 解析 152 维技术特征
technical_features = collector._parse_technical_features(match_data)

# 存储到数据库
import psycopg2
conn = psycopg2.connect(...)
cursor = conn.cursor()
cursor.execute("""
    UPDATE matches
    SET technical_features = %s
    WHERE match_id = %s
""", (json.dumps(technical_features), match_id))
conn.commit()
```

### xG 查询示例

```sql
-- 查询 xG 数据
SELECT
    match_id,
    home_team,
    away_team,
    technical_features->>'home_xg' as home_xg,
    technical_features->>'away_xg' as away_xg,
    technical_features->>'total_xg' as total_xg
FROM matches
WHERE technical_features->>'home_xg' IS NOT NULL
ORDER BY match_time DESC
LIMIT 10;
```

### 按赛季统计 xG 覆盖率

```sql
-- 按赛季统计 xG 覆盖率
SELECT
    season,
    COUNT(*) as total_matches,
    COUNT(technical_features->>'home_xg') as with_xg,
    ROUND(COUNT(technical_features->>'home_xg') * 100.0 / COUNT(*), 2) as xg_coverage
FROM matches
GROUP BY season
ORDER BY season;
```
