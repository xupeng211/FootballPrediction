### 2. Playwright API 拦截

```python
async def capture_league_data(url: str):
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=True)
        page = await browser.new_page()

        captured = None
        async def handle_response(response):
            if 'ajax-sport-country-tournament-archive' in response.url:
                body = await response.body()
                if len(body) > 100KB:
                    captured = body

        page.on('response', handle_response)

        await page.goto(url, timeout=90000)
        await asyncio.sleep(2)

        # 滚动触发 API
        for _ in range(10):
            await page.mouse.wheel(0, 500)
            await asyncio.sleep(0.2)

        return captured
```

```python
def two_stage_matching(oddsportal_matches, db_matches, threshold=80.0):
    # 第一阶段：规范化匹配
    normalized_op = {
        (normalize(m['home']), normalize(m['away'])): m

    for db_id, db_home, db_away in db_matches:
```

## 模块化执行问题分析

### 问题现象
```bash
# ❌ 失败 - 脚本执行
$ python scripts/v134_0_full_harvest.py --league "Premier League" --season "23/24"
Page.goto: Timeout 90000ms exceeded

# ✅ 成功 - 内联执行
$ python3 -c "<code>"
✅ 捕获: 964,796 字节
```

### 根本原因
1. **sys.path.insert 与 Playwright 冲突**:
   - 在脚本顶部添加 `sys.path.insert(0, str(Path(__file__).parent.parent))`
   - 导致 Playwright 的异步事件循环行为异常

2. **导入顺序问题**:
   - `src` 模块导入与 `playwright` 导入的相对顺序
   - 影响事件循环初始化

3. **响应体访问错误**:
   - `Protocol error (Network.getResponseBody): No data found`
   - 响应体被多次访问导致冲突

### 解决方案
**方案 A**: 使用内联代码执行（当前可用）
```bash
python3 -c "
import asyncio, sys
sys.path.insert(0, '.')
from playwright.async_api import async_playwright
# ... 完整代码 ...
"
```
