### 增强的错误分类系统

**新增 `FetchError` 枚举类**（9种错误类型）:

```python
class FetchError(Enum):
    NETWORK_ERROR = "network_error"
    TIMEOUT_ERROR = "timeout_error"
    HTTP_ERROR = "http_error"
    JSON_DECODE_ERROR = "json_decode_error"
    JSON_STRUCTURE_ERROR = "json_structure_error"
    MATCH_CANCELLED = "match_cancelled"
    MATCH_POSTPONED = "match_postponed"
    LEAGUE_NOT_FOUND = "league_not_found"
    RATE_LIMITED = "rate_limited"
    UNKNOWN_ERROR = "unknown_error"
```

### 增强的异常处理机制

**核心改进点**:

1. **细致的错误分类**
   - 网络错误 vs 超时错误
   - HTTP 429 限流检测
   - JSON 解码失败 vs 结构错误
   - 比赛取消/延期特殊情况

2. **结构化日志记录**
   ```python
   log_entry = {
       "timestamp": "2026-01-16T21:55:23",
       "match_id": 12345,
       "error_type": "network_error",
       "error_message": "Connection refused",
       "collector_version": "V41.114",
   }
   ```

3. **优雅降级处理**
   - 比赛取消 → 仍提取基础数据
   - JSON 结构缺失 → 使用 `.get()` 安全访问
   - API 限流 → 自动重试机制

### 特殊情况处理

**比赛取消/延期场景**:
```python
def _extract_match_status(self, json_data: dict) -> MatchStatus:
    """提取比赛状态，支持取消/延期检测"""
    status_code = json_data.get("header", {}).get("status", {}).get("reason", {}).get("short", "")

    if status_code == "CAN":
        return MatchStatus.CANCELLED
    elif status_code == "PST":
        return MatchStatus.POSTPONED
    # ...
```

**基础数据提取（即使比赛取消）**:
```python
def _extract_basic_data_safe(self, json_data: dict, match_id: int) -> dict:
    """安全提取基础数据，容错处理"""
    basic_data = {
        "match_id": match_id,
        "extraction_status": "partial",  # 标记为部分提取
        "home_team": json_data.get("header", {}).get("teams", [{}])[0].get("name"),
        "away_team": json_data.get("header", {}).get("teams", [{}])[1].get("name"),
        # ... 使用 .get() 避免 KeyError
    }
    return basic_data
```
