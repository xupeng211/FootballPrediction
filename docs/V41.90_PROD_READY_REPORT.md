## ⚠️ 灰度测试失败分析

### 失败原因

```
Cannot connect to host api.ipify.org:443 ssl:default [None]
```

**根本原因**: 网络连接失败，无法通过代理访问外部 API

**可能原因**:
1. **代理服务未启动**: 宿主机代理客户端可能未运行
2. **防火墙阻止**: Windows 防火墙可能阻止了 WSL2 访问
3. **代理配置错误**: 代理端口或地址配置不正确
4. **DNS 解析失败**: 无法解析 api.ipify.org

### V41.87 代理审计回顾

根据 V41.87 的审计结果：
- **可用端口**: 5/19 (26.3%)
- **可用端口列表**: 7891, 7892, 7893, 7894, 7895
- **问题**: 14 个端口返回 ClientConnectorError

**关联性**: 灰度测试使用的是 7891 端口，这是可用端口之一，说明问题可能是：
- 代理服务在 V41.87 审计后被关闭
- 网络环境发生变化
- Windows 防火墙策略更新

## 🚀 全量巡航建议参数

### 前置条件 (必须满足)

- [x] ✅ 数据库健康度: 良好 (2/4 项通过)
- [ ] ❌ 代理服务可用: 需验证

### 推荐巡航参数 (网络修复后)

#### 阶段 1: 保守模式 (前 100 场)

```bash
export DOCKER_ENV=true
python main.py --source fotmob --mode single \
  --league "Premier League" \
  --season "23/24" \
  --limit 100 \
  --no-ghost
```

**参数说明**:
- `--source fotmob`: 使用 FotMob API (更稳定)
- `--mode single`: 单次收割模式
- `--league "Premier League"`: 仅限英超
- `--season "23/24"`: 指定赛季
- `--limit 100`: 限制 100 场
- `--no-ghost`: 禁用 Ghost Protocol (降低指纹复杂度)

**预期效果**:
- 采集时间: ~30 分钟
- 成功率: >80%
- IP 封禁风险: 低

---

#### 阶段 2: 扩展模式 (100-500 场)

```bash
export DOCKER_ENV=true
python main.py --source fotmob --mode single \
  --league "Premier League" \
  --season "23/24" \
  --limit 500 \
  --proxy-file scripts/config/5_available_ports.txt
```

**新增参数**:
- `--limit 500`: 扩展到 500 场
- `--proxy-file`: 指定 5 个可用端口列表

**创建端口配置文件**:
```bash
cat > scripts/config/5_available_ports.txt << EOF
http://172.25.16.1:7891
http://172.25.16.1:7892
http://172.25.16.1:7893
http://172.25.16.1:7894
http://172.25.16.1:7895
EOF
```

---

#### 阶段 3: 全量巡航 (所有联赛)

```bash
export DOCKER_ENV=true
python main.py --source fotmob --mode cruise \
  --interval 3600
```

**参数说明**:
- `--mode cruise`: 24h 全自动巡航
- `--interval 3600`: 每小时执行一次

**适用条件**:
- 阶段 1 和 2 成功率 > 90%
- IP 封禁率 < 5%
- 数据库写入正常

### 代理配置参考

#### Clash 配置
```yaml
# external-controller: 127.0.0.1:9090
# secret: ""

# Allow LAN: 设置 allow-lan 为 true
allow-lan: true
bind-address: '0.0.0.0'
mode: rule
log-level: info
external-controller: 0.0.0.0:9090

# HTTP 代理端口
port: 7891

# SOCKS5 代理端口
socks-port: 7892

# Mixed 端口 (HTTP + SOCKS5)
mixed-port: 7893
```
