# Model Artifacts

## Purpose

Model artifacts are generated outputs and are not stored in Git.

## Not Stored in Git

The following are intentionally excluded from Git:

- `models/`
- `model_zoo/`
- `*.joblib`
- `*.pkl`
- `*.onnx`
- `*.pt`
- `*.pth`
- `*.h5`
- `*.ckpt`

## Local Development

For local development, model files should be restored from external artifact storage or regenerated by the training pipeline.

Recommended local paths:

- `models/`
- `model_zoo/`

These paths are ignored by Git.

## CI / Tests

Tests should not require production model artifacts unless they are explicitly integration tests.

Unit tests should use:

- small fixtures
- mocks
- synthetic models
- generated temporary artifacts

## Recovery / Generation

If no external artifact storage is configured yet, this project must define one before relying on fresh clones for prediction workloads.

Recommended next steps:

1. Define canonical model artifact storage.
2. Add a model download or generation script.
3. Document required model filenames and checksums.
4. Ensure CI does not depend on local-only model files.
