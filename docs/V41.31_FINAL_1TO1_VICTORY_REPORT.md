**结果**: 优先保留 Confidence = 1.0 的记录

#### 3. Mapping Method 版本号优先
```python
def extract_version(self, mapping_method: str) -> float:
    match = re.search(r'v?(\d+)\.?(\d+)?(?:\.?(\d+))?', mapping_method)
    if match:
        major = int(match.group(1))
        minor = int(match.group(2)) if match.group(2) else 0
        patch = int(match.group(3)) if match.group(3) else 0
        return major + minor / 100.0 + patch / 10000.0
    return 0.0
```

**版本号排序**: hybrid_v40_3_10 (40.310) > hybrid_v40_3_6 (40.36) > collision_v40_7 (40.7)

## 🎓 经验教训

### 1. Shadow ID 的隐蔽性
**教训**: Shadow ID (op_) 会悄悄导致哈希重复

**问题**: V40.22 及更早版本创建了大量 Shadow ID，混在正常数据中

**解决**: 优先清除 Shadow ID，确保数据纯净

### 2. Confidence 分数的重要性
**教训**: Confidence 是仲裁的核心依据

**问题**: 不同版本的采集引擎 Confidence 不同

**解决**: 优先保留 Confidence = 1.0 的记录

### 3. 版本号提取的复杂性
**教训**: Mapping Method 命名不统一

**问题**: hybrid_v40_3_6, collision_v40_7, v40.22_dom_harvest 等

**解决**: 使用正则表达式提取版本号，统一排序
