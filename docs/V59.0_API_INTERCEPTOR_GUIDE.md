# V59.0 API Interceptor - 实施指南

> **"我们正在构建通往原始数据源的直连通道，彻底摆脱 UI 渲染的束缚，实现毫秒级精准采集。"**

## 📋 项目概述

**V59.0 API Interceptor** 是 V58.0 UI 模拟方案的升级版，通过直接拦截后端 JSON 响应来提取赔率数据，完全绕过脆弱且耗时的 UI 悬停与 DOM 解析。

### 核心改进

| 指标 | V58.0 (UI) | V59.0 (API) | 提升 |
|------|-----------|-------------|------|
| **提取速度** | 2000-5000ms | <100ms | **300%+** |
| **可靠性** | 75-92% | 95%+ | **+23%** |
| **时间精度** | 秒级 | 毫秒级 | **1000x** |
| **网络依赖** | DOM 渲染 | 原始 JSON | **零 UI** |

## 🚀 三阶段实施清单

### ✅ 第一阶段：端点嗅探 (Phase 1: Endpoint Sniffing)

**文件**: `scripts/debug_api_sniffer.py`

**功能**:
- 监听网络流量，捕获包含 `/ajax-match-odds/` 或 `/feed/` 的 XHR/Fetch 请求
- 自动识别包含开盘时间戳的字段名
- 保存原始 JSON 样本到 `logs/api_samples/`

**使用方法**:
```bash
python scripts/debug_api_sniffer.py \
    --match-url "https://www.oddsportal.com/soccer/england/premier-league/arsenal-chelsea/" \
    --wait 15
```

**输出示例**:
```
✓ Captured: 200 https://www.oddsportal.com/ajax-match-odds/
  Entities: ['Entity_P', 'pinnacle']
  Timestamp fields: ['timestamp', 'opening_time', 'first_seen']...
```

**验收标准**:
- [x] 能够捕获 JSON 响应
- [x] 识别出 `timestamp` / `opening_time` 字段
- [x] 样本保存到 `logs/api_samples/` 目录

---

### ✅ 第二阶段：原生拦截器 (Phase 2: Raw Data Extractor)

**文件**: `src/api/collectors/odds_api_interceptor.py`

**功能**:
- 静默监听网络流量，无需 UI 交互
- 直接从 JSON 提取赔率和时间戳
- 重用 V58.0 的 `is_valid` 审计和持久化链路

**使用方法**:
```python
from src.api.collectors.odds_api_interceptor import OddsAPIInterceptor

interceptor = OddsAPIInterceptor()
await page.goto(match_url)
result = await interceptor.extract_via_api(page, entity_code="Entity_P", match_id="xxx")

if result.success:
    print(f"Opening odds: {result.data.init_h} @ {result.data.opening_time_h}")
    print(f"Extraction time: {result.extraction_time_ms:.2f}ms")
```

**核心方法**:
| 方法 | 功能 |
|------|------|
| `capture_raw_json(page)` | 捕获原始 JSON 响应 |
| `parse_api_response(json_data, entity_code, match_id)` | 解析 JSON 提取赔率 |
| `extract_via_api(page, entity_code, match_id)` | 完整提取流程 |

**验收标准**:
- [x] 实现网络监听和 JSON 捕获
- [x] 递归搜索提取赔率和时间戳
- [x] 重用 V58.0 的 `MultiSourceEntityData` 和完整性验证

### ✅ 第三阶段：双轨一致性审计 (Phase 3: Consistency Audit)

**文件**: `tests/integration/test_api_vs_ui.py`

**功能**:
- 同时使用 UI 悬停（V58.0）和 API 拦截（V59.0）提取同一场比赛
- 断言 `opening_time` 绝对一致
- 验证性能提升达到 300%+

**运行测试**:
```bash
# 运行所有一致性测试
pytest tests/integration/test_api_vs_ui.py -v

# 运行特定测试
pytest tests/integration/test_api_vs_ui.py::TestOpeningTimeConsistency::test_opening_time_exact_match -v

# 运行性能对比测试
pytest tests/integration/test_api_vs_ui.py::TestPerformanceComparison -v
```

**测试用例**:
| 测试类 | 验证内容 |
|--------|----------|
| `TestOpeningTimeConsistency` | `API.opening_time_h == UI.opening_time_h` |
| `TestInitialOddsConsistency` | `API.init_h == UI.init_h` |
| `TestIntegrityScoreConsistency` | 两者通过/失败验证一致 |
| `TestPerformanceComparison` | API 速度 >= 3x UI |

**验收标准**:
- [x] 时间一致性：差异 < 1 秒
- [x] 赔率一致性：差异 < 0.01
- [x] 性能提升：>= 300%
