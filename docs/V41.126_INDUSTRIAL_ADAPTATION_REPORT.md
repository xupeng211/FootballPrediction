### 1. 别名库自动注浆系统 (P0 - 扩展性核心)

**设计理念**: 当对齐分数 ≥ 0.95 (HIGH 置信度) 时，自动将队伍映射关系写入 `alias_teams` 表，实现"自我学习、自我完善"。

#### 表结构设计

```sql
CREATE TABLE alias_teams (
    id SERIAL PRIMARY KEY,
    fotmob_team_id VARCHAR(50) NOT NULL,
    fotmob_team_name VARCHAR(255) NOT NULL,
    oddsportal_team_name VARCHAR(255),
    oddsportal_team_path VARCHAR(500),

    -- 置信度追踪
    confidence FLOAT DEFAULT 0.95,
    alignment_count INTEGER DEFAULT 1,
    last_aligned_at TIMESTAMP DEFAULT NOW(),
    created_at TIMESTAMP DEFAULT NOW(),

    -- 元数据
    league_name VARCHAR(255),
    is_active BOOLEAN DEFAULT TRUE,

    -- 唯一约束
    UNIQUE(fotmob_team_id, oddsportal_team_name)
);
```

#### 自动注浆逻辑

```python
def _inject_alias_mapping(
    self,
    fotmob_match: dict[str, Any],
    oddsportal_match: dict[str, Any],
    score: float,
    confidence: str
) -> bool:
    """
    V41.126: 自动注浆别名库

    当对齐分数 ≥ 0.95 (HIGH 置信度) 时，自动将队伍映射关系写入 alias_teams 表。
    """
    # 只在 HIGH 置信度时自动注浆
    if score < 0.95 or confidence != "HIGH":
        return False

    # 使用 ON CONFLICT DO UPDATE 实现幂等性
    # 重复对齐时递增 alignment_count，保留最高 confidence
```

**关键特性**:
- ✅ **幂等性**: `ON CONFLICT DO UPDATE` 确保重复对齐时递增计数而非重复插入
- ✅ **置信度递增**: `GREATEST(alias_teams.confidence, EXCLUDED.confidence)` 保留最高置信度
- ✅ **时间戳追踪**: `last_aligned_at` 记录最近一次对齐时间
- ✅ **联赛上下文**: `league_name` 支持按联赛组织别名

### 2. 动态调权机制 (P0 - 解决未开赛难题)

**问题**: V41.125 的固定权重（50% 队名 + 30% 比分 + 10% 时间 + 10% ID）在未开赛时无法使用（无比分数据），导致最高只能得 0.6 分，无法达到 0.85 阈值。

**解决方案**: 根据比赛状态动态调整权重和阈值。

#### 比赛状态检测

```python
def _is_match_live_or_finished(self, fotmob_match: dict[str, Any]) -> bool:
    """
    V41.126: 判断比赛是否正在进行或已完场

    方法 1: 检查比分数据（非零比分 → 已开赛）
    方法 2: 检查比赛时间（过去时间 → 已开赛）
    """
    # 方法 1: 检查比分数据
    score_str = fotmob_match.get("score_str", "").strip()
    if score_str and score_str not in ["", "0:0", "0-0", "-"]:
        return True  # 有非零比分，说明已开赛

    # 方法 2: 检查比赛时间
    match_date = fotmob_match.get("match_date")
    if match_date:
        if datetime.now() > match_date:
            return True  # 比赛时间已过，说明已开赛

    return False  # 默认认为是未开赛
```

#### 动态权重策略

| 比赛状态 | 队名权重 | 时间权重 | 阈值 | 最高分 |
|----------|----------|----------|------|--------|
| **已开赛/已完场** | 50% | 10% | 0.85 | 1.0 (50+30+10+10) |
| **未开赛** | 80% | 20% | 0.80 | 1.0 (80+20) |

**未开赛权重优化**:
- ✅ **队名权重提升**: 50% → 80%（无比分时依赖队名）
- ✅ **时间权重提升**: 10% → 20%（放宽时间窗口到 48h）
- ✅ **阈值降低**: 0.85 → 0.80（适应无比分场景）

#### API 使用

```python
# 统一入口 - 自动选择权重策略
result = service.align_with_dynamic_weighting(
    fotmob_match=fotmob_match,
    oddsportal_match=oddsportal_match,
    verbose=True,
    auto_inject=True  # 自动注浆别名库
)

# 返回结果包含 match_status 标记
result["match_status"]  # "LIVE_OR_FINISHED" 或 "UPCOMING"
result["threshold"]     # 0.85 (已开赛) 或 0.80 (未开赛)
```

## 🛡️ 风险与限制

### 已知限制

1. **别名库污染风险**: 如果初始对齐错误，错误映射会被固化
   - **缓解**: 只在 ≥0.95 (HIGH 置信度) 时注浆，设置高阈值

2. **未开赛准确率**: 无比分时准确率可能略低（80% 阈值）
   - **缓解**: 开赛后使用标准权重重新验证

3. **联赛特异性**: 不同联赛的队名命名规则可能不同
   - **缓解**: `league_name` 字段支持按联赛组织别名

### 误报场景

| 场景 | 风险 | 缓解措施 |
|------|------|----------|
| 同一球队短时间内有两场比赛 | 可能关联错误比赛 | 时间窗口 + ID 映射双重验证 |
| 队名拼写相似但不同球队 | 可能误关联 | 80% 队名阈值确保高相似度 |
| 延迟开球 > 48h | 未开赛可能被拒绝 | 人工复核或放宽时间窗口 |
