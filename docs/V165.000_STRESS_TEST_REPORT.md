## 二、根本原因分析

### 2.1 数据流发现

通过深度诊断，发现了 **OddsPortal 的实际数据加载机制**：

```
┌─────────────────────────────────────────────────────────────┐
│  1. 页面加载                                                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  2. XHR 请求: /match-event/1-1-jVjHFI03-1-2-*.dat           │
│     • 响应大小: 18536 bytes                                  │
│     • Content-Type: application/json                         │
│     • 实际格式: Base64 编码字符串                             │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 客户端解码                                                │
│     Base64 解码 → JXG.decompress → JSON.parse                │
│     M2s4dGczUUtN... → 3k8tg3QKM... → {odds data}            │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 内存存储 (Vue/React state)                               │
│     window.__STATE__ or component state                     │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 钩子失效原因

**`apex_engine.js` 的钩子设计存在盲点**：

```javascript
// 当前钩子只能拦截明文 JSON.parse
JSON.parse = function(text, reviver) {
    const result = originalJSONParse.call(this, text, reviver);
    mountOddsData(result, 'JSON.parse');  // ❌ 无法触发
    return result;
};
```

**问题**：
1. `/match-event/` 响应是 **Base64 字符串**，不是 JSON 对象
2. 客户端先调用 `atob()` 解码，再调用 `JXG.decompress()` 解压
3. 最后才调用 `JSON.parse()` 解析，但此时数据已嵌套在回调中
4. `isOddsData()` 模式匹配无法识别解压后的数据结构

### 2.3 验证数据格式

解码后的数据片段：
```
3k8tg3QKMh1kQ+fdOmJyxfE15bc4D67aymGT0XRX/JryBj51NZB96nhnTn9vtcmy3UV6G...
```

这是 **JXG 库压缩后的数据**，需要使用 `JXG.Constants.decompress()` 解压。
