## 🔒 Error Handling 核心代码

### 1. 熔断器 (Circuit Breaker)

```python
from src.ml.fault_tolerance import CircuitBreaker, CircuitBreakerConfig

# 配置熔断器
config = CircuitBreakerConfig(
    failure_threshold=20,        # 连续 20 次 403 错误触发熔断
    timeout_seconds=300,         # 5 分钟后尝试恢复
    error_codes={403, 429, 500, 502, 503}
)

circuit_breaker = CircuitBreaker(config)

# 在处理循环中
if not circuit_breaker.can_execute():
    logger.error("熔断器已开启，停止处理")
    send_alert("Circuit breaker tripped!")
    sys.exit(1)

try:
    result = process_match(match_id)
    circuit_breaker.record_success()
except APIError as e:
    if e.status_code == 403:
        circuit_breaker.record_failure(error_code=403)
```

### 2. 断点续传 (Checkpoint Tracker)

```python
from src.ml.fault_tolerance import CheckpointTracker

tracker = CheckpointTracker("data/backfill_progress.json")
tracker.set_total(total_matches)

for match_id in match_ids:
    # 检查是否已处理
    if tracker.is_processed(match_id):
        continue

    try:
        result = process_match(match_id)
        tracker.record_success(match_id)
    except Exception as e:
        tracker.record_failure(match_id)

# 获取进度
progress = tracker.get_progress()
print(f"进度: {progress['processed']}/{progress['total_matches']}")
print(f"成功率: {progress['success_rate']*100:.1f}%")
```

```bash
# 1. 创建回填脚本
cat > src/ops/run_backfill_v20.5.sh << 'EOF'
#!/bin/bash
set -euo pipefail

# 加载环境
source .venv/bin/activate

# 启动回填（带容错）
python -m src.ml.fault_tolerance_backfill \
    --manifest data/production/manifest_54_2324.csv \
    --workers 4 \
    --delay 2.0 \
    --checkpoint data/backfill_progress.json \
    --max-retries 2

EOF

chmod +x src/ops/run_backfill_v20.5.sh

# 2. 运行
./src/ops/run_backfill_v20.5.sh
```
