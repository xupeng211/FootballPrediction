## 🎓 经验教训

### 1. NULL 值的隐蔽性
**教训**: NULL 值会悄悄导致数据统计错误

**问题**:
- 518 条记录的 league_name 为 NULL
- WHERE league_name = 'Premier League' 排除了这些记录
- 导致覆盖率统计严重偏低

**解决**: 定期检查 NULL 值，使用 COALESCE 或 IS NULL 条件

### 2. SQL JOIN 的陷阱
**教训**: LEFT JOIN 条件必须精确匹配

**问题**:
```sql
-- ❌ 错误：条件太严格
LEFT JOIN matches_mapping mm
    ON m.match_id = mm.fotmob_id
    AND m.season = '2023/2024'  -- 如果 mm.season 为 NULL，匹配失败
    AND m.league_name = mm.league_name  -- 如果 mm.league_name 为 NULL，匹配失败
```

**解决**:
```sql
-- ✅ 正确：使用 NOT EXISTS
WHERE NOT EXISTS (
    SELECT 1 FROM matches_mapping mm
    WHERE mm.fotmob_id = m.match_id
)
```
