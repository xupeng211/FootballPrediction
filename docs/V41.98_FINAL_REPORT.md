## 2️⃣ 环境验证函数集成 (P0) ✅

### `verify_docker_environment()` 实现

**文件**: `src/database/connection.py`

**新增功能**:
1. **异步验证函数** (`verify_docker_environment()`)
   - 检查 PostgreSQL 版本（必须是 PostgreSQL 15）
   - 检查数据库名称（必须是 football_db）
   - 检查服务器地址

2. **DatabaseManager 新方法**:
   - `initialize_and_verify()`: 初始化并验证环境
   - `verify_environment_async()`: 独立验证环境

3. **模块级便捷函数**:
   - `initialize_database_and_verify()`: 全局异步初始化
   - `verify_database_environment()`: 全局环境验证

**验证逻辑**:
```python
async def verify_docker_environment(engine: AsyncEngine) -> bool:
    """V41.98: 验证数据库连接是否指向 Docker 容器"""
    async with engine.begin() as conn:
        # 检查 PostgreSQL 版本
        result = await conn.execute(text("SELECT version()"))
        version = result.scalar()

        # 检查数据库名称
        result_db = await conn.execute(text("SELECT current_database()"))
        db_name = result_db.scalar()

    # 环境纯净性检查
    is_postgres_15 = "PostgreSQL 15" in version
    is_correct_db = db_name == "football_db"
    is_docker = is_postgres_15 and is_correct_db

    if not is_docker:
        raise RuntimeError(
            f"❌ V41.98 环境验证失败!\n"
            f"   当前数据库: {db_name}\n"
            f"   PostgreSQL 版本: {version[:50]}...\n"
            f"   期望: PostgreSQL 15 + football_db (Docker 容器)\n"
        )

    return True
```

**使用示例**:
```python
# 方式 1: 使用带验证的初始化
from src.database.connection import initialize_database_and_verify
await initialize_database_and_verify()

# 方式 2: 先初始化，后验证
from src.database.connection import initialize_database, verify_database_environment
initialize_database()
await verify_database_environment()
```
