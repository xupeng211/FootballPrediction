## 💡 经验教训

### 1. API 端点差异导致 match_id 不匹配

**问题**: 不同 FotMob API 端点返回不同格式的 match_id

**解决**:
- 使用现有 match_id 直接查询
- 通过 `FotMobCoreCollector.fetch_match_details()` 获取详情

### 2. API 响应结构解析

**问题**: FotMob API 的 team 数据在 `header.teams[]` 数组，而非 `home.id`/`away.id`

**解决**:
```python
teams = header.get("teams", [])
home_team_id = str(teams[0].get("id"))
away_team_id = str(teams[1].get("id"))
```

### 3. UPSERT vs INSERT

**问题**: match_id 不匹配时，UPSERT 退化为 INSERT

**解决**:
- V41.120 直接使用 `UPDATE ... WHERE match_id = %s`
- 避免创建新记录

## 🏆 最佳实践总结

### 数据回补标准流程

```
1. 检查现有数据 → 定位缺失字段
2. 提取现有 ID → 使用数据库中的 match_id
3. API 精准查询 → fetch_match_details(match_id)
4. 正确解析响应 → 按实际 API 结构解析
5. 直接更新记录 → UPDATE 而非 UPSERT
6. 验证结果 → 检查覆盖率和数据完整性
```

- ❌ **禁止使用不同端点的 match_id** 进行 UPSERT
