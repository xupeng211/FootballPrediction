# TDD代码审查检查清单

## 📋 代码审查前的准备

### 审查者准备
- [ ] 理解功能需求和用户故事
- [ ] 阅读相关测试用例
- [ ] 熟悉TDD最佳实践

### 开发者准备
- [ ] 确保所有测试通过
- [ ] 代码覆盖率达标（≥30%）
- [ ] 准备功能说明文档

## 🔍 代码审查检查点

### 1. 测试质量检查

#### 测试设计
- [ ] **测试先于实现**：功能代码前已有测试
- [ ] **测试覆盖完整**：包括正常、边界、错误情况
- [ ] **测试命名清晰**：描述测试场景和期望
- [ ] **测试独立运行**：不依赖其他测试状态

#### 测试实现
```python
# ✅ 好的测试示例
def test_user_registration_success():
    """测试用户注册成功流程"""
    # Arrange - 准备测试数据
    user_data = {
        "username": "testuser",
        "email": "test@example.com",
        "password": "securepassword123"
    }

    # Act - 执行操作
    user = UserService().register(user_data)

    # Assert - 验证结果
    assert user.username == "testuser"
    assert user.is_active is True
    assert user.id is not None
```

```python
# ❌ 避免的反模式
def test_something():
    # 直接调用实现，没有准备步骤
    result = complex_function()
    assert result == "expected"
```

### 2. 代码质量检查

#### Red阶段检查
- [ ] 测试确实先于代码实现
- [ ] 测试失败原因明确（不是语法错误）
- [ ] 测试失败消息清晰

#### Green阶段检查
- [ ] 代码刚好让测试通过，没有多余实现
- [ ] 代码逻辑简单直接
- [ ] 没有硬编码的测试数据

#### Refactor阶段检查
- [ ] 在测试保护下进行重构
- [ ] 消除了重复代码
- [ ] 提高了代码可读性
- [ ] 保持了所有测试通过

### 3. 设计质量检查

#### 单一职责
- [ ] 每个函数只做一件事
- [ ] 类职责明确
- [ ] 模块内聚性高

#### 依赖管理
- [ ] 依赖关系清晰
- [ ] 使用接口而非具体实现
- [ ] 避免循环依赖

#### 错误处理
- [ ] 有适当的错误处理
- [ ] 错误信息有意义
- [ ] 异常类型合理

### 4. TDD流程检查

#### 是否遵循TDD
```python
# ✅ 遵循TDD的代码结构
class Calculator:
    """先有测试，后有实现"""
    def add(self, a: int, b: int) -> int:
        """功能简单，刚好满足测试需求"""
        return a + b

# 对应的测试
def test_calculator_add():
    calc = Calculator()
    assert calc.add(2, 3) == 5
```

#### 测试驱动证据
- [ ] Git提交显示测试先于实现
- [ ] PR中有测试覆盖历史
- [ ] 可以看到Red-Green-Refactor的迭代

## 📊 代码审查评分

### 评分标准（满分10分）

| 维度 | 评分标准 | 得分 |
|------|----------|------|
| 测试设计 | 测试先于实现，覆盖完整 | 0-3 |
| 代码质量 | 代码简洁，设计良好 | 0-2 |
| TDD流程 | 遵循TDD原则 | 0-3 |
| 文档说明 | 代码有清晰的注释和文档 | 0-1 |
| 整体评估 | 代码质量高，可维护性强 | 0-1 |

### 评级
- **优秀**：8-10分
- **良好**：6-7分
- **合格**：4-5分
- **需改进**：0-3分

## 🔄 持续改进

### 团队改进措施
1. **定期培训**：每月TDD最佳实践分享
2. **代码审查培训**：提升审查者的TDD识别能力
3. **质量门禁**：强制要求测试覆盖率
4. **工具支持**：使用自动化工具检查TDD遵循情况

### 个人提升建议
1. **阅读经典**：《测试驱动开发》Kent Beck
2. **实践练习**：每天使用TDD开发一个小功能
3. **参与社区**：分享TDD经验和教训
4. **持续学习**：关注TDD最新发展
