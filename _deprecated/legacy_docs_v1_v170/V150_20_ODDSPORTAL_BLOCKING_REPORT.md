# V150.20 OddsPortal 拦截诊断报告

**日期**: 2026-01-09
**状态**: ❌ **完全拦截 - 需要更换采集方案**

---

## 📋 执行摘要

### 问题发现

V150.20 深度 DOM 审计发现 **OddsPortal 已完全拦截了我们的所有访问请求**。

### 证据

| 证据 | 说明 | 结果 |
|------|------|------|
| **HTML 内容** | `<html><head></head><body></body></html>` | ❌ 完全空白 |
| **页面截图** | 白屏，无任何内容 | ❌ 无数据 |
| **Body 长度** | 0 字符 | ❌ 空页面 |
| **所有策略** | 4 种绕过策略全部失败 | ❌ 全部拦截 |

---

## 🛡️ 绕过策略尝试结果

### 策略 1: 原生浏览器（无 stealth）
```
配置: Chromium, headless=True, 禁用自动化检测
结果: ❌ Body 长度 0
```

### 策略 2: 可见浏览器（非 headless）
```
配置: Chromium, headless=False
结果: ❌ ERR_HTTP_RESPONSE_CODE_FAILURE
```

### 策略 3: 拟人化导航（从首页开始）
```
配置: 先访问首页 → 滚动 → 再访问目标页面
结果: ❌ Body 长度 0
```

### 策略 4: 代理 + 延迟
```
配置: 代理 7890 + 20 秒延迟
结果: ❌ Body 长度 0
```

## 🔍 根本原因分析

### 可能原因

| 原因 | 可能性 | 说明 |
|------|--------|------|
| **IP 封禁** | 🔴 高 | IP 地址被永久封禁 |
| **TLS/JA3 指纹** | 🔴 高 | TLS 指纹被识别为爬虫 |
| **WSL2 环境特征** | 🟡 中 | WSL2 特征被检测 |
| **User-Agent 检测** | 🟢 低 | UA 检测（已伪装） |

### 历史记录回顾

| 版本 | 时间 | 结果 | 说明 |
|------|------|------|------|
| V150.18.1 | 2026-01-08 02:00 | ✅ 部分成功 | 页面加载，但 0 数据点 |
| V150.19 | 2026-01-09 02:11 | ✅ 部分成功 | 页面加载，但 0 数据点 |
| V150.20 | 2026-01-09 02:22 | ❌ 完全失败 | 空白页面 |

**结论**: 在 V150.19 和 V150.20 之间，OddsPortal 加强了反爬虫措施，现在完全拦截我们的访问。

## 🔄 建议方案

### 短期方案（1-2 周）

1. **暂停 OddsPortal 采集**
   - 标记所有 OddsPortal 相关任务为"不可用"
   - 保存当前状态以便将来恢复

2. **专注 FotMob 数据源**
   - V144.5 FotMob API 采集器继续工作
   - 扩大 FotMob 数据采集范围（V26.6 全球联赛）

3. **数据库清理**
   - 删除 319 条错误映射记录
   - 保留 69 条正确映射记录

### 中期方案（1-2 月）

1. **更换 IP 地址**
   - 使用住宅代理
   - 轮换多个出口 IP

2. **浏览器指纹优化**
   - 使用 Playwright 的 `device_descriptors` 模拟真实设备
   - 考虑使用 `browserless` 或类似服务

3. **降低采集频率**
   - 从实时采集改为每小时一次
   - 使用指数退避策略

### 长期方案（3-6 月）

1. **寻找替代数据源**
   - 调研其他赔率数据提供商
   - 评估数据质量和成本

2. **商业数据采购**
   - 考虑购买官方数据 API
   - 评估 ROI（投入产出比）

3. **自建数据采集网络**
   - 使用分布式采集节点
   - 使用不同地区的服务器
