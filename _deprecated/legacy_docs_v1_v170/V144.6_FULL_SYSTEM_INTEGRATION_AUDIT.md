## 🎯 Task B: 核心功能"闭环"审计 (Functional Audit)

### B.1 数据源对齐 - main.py 统一入口

#### main.py 版本信息
```python
# main.py (V142.0)
"""
Features:
    - Automatic environment pre-check (WSL2 proxy discovery, IP detection)
    - Mode scheduling (single/cruise/check)
    - Rich console output with professional formatting
    - Integration with all Phase 1 components (BaseExtractor V141.0, TeamNameNormalizer V140.0)

Usage:
    python main.py --mode single --league "Premier League" --season "23/24"
    python main.py --mode cruise
    python main.py --mode check
"""
```

#### ❌ 关键发现：缺少 `--source` 参数

**当前参数**:
```bash
python main.py --mode {single|cruise|check} [其他参数]
```

**缺失功能**:
```bash
python main.py --source fotmob --mode single    # ❌ 不支持
python main.py --source oddsportal --mode cruise # ❌ 不支持
```

**影响**: 无法通过 main.py 切换数据源

**根因**: `main.py` (V142.0) 设计为统一入口，但未实现数据源参数

### B.2 Schema 最终一致性

#### 数据库 Schema (matches 表)
```sql
Total columns: 33 (not 35 as expected)

V36.0 字段 ✅:
- season_id        VARCHAR(10)    NULL
- season_name      VARCHAR(20)    NULL
- match_time_utc   VARCHAR(50)    NULL
- fetched_at       TIMESTAMP      NULL
```

#### L1 Pydantic Schema (l1_match_schema.py V36.0)
```python
class L1MatchData(BaseModel):
    # 核心标识
    match_id: str
    league_id: str
    league_name: str

    # 赛季信息 (V36.0)
    season_id: str
    season_name: str

    # 球队信息
    home_team: str
    away_team: str
    home_team_id: int
    away_team_id: int

    # 比赛状态
    status: MatchStatus
    match_time_utc: str

    # 比分（可选）
    home_score: int | None
    away_score: int | None

    # 元数据
    fetched_at: str
```

**字段对比**:
| 类别 | 数据库列数 | L1 Schema 字段 | 状态 |
|------|-----------|---------------|------|
| **V36.0 新增字段** | 4 | 4 | ✅ 完全对齐 |
| **总字段数** | 33 | 13 | ⚠️ 部分覆盖 |

**结论**: L1 Schema 只定义核心字段，不包含所有数据库列。这是**设计如此**，不是缺陷。
