### 1. 数据库约束系统

```sql
-- 部分唯一索引（只对非 NULL 8 位哈希生效）
CREATE UNIQUE INDEX idx_matches_mapping_oddsportal_hash_unique
ON matches_mapping (oddsportal_hash)
WHERE oddsportal_hash IS NOT NULL
  AND LENGTH(oddsportal_hash) = 8;

-- 触发器（自动检测重复哈希）
CREATE TRIGGER trg_check_unique_hash_before_insert
    BEFORE INSERT OR UPDATE OF oddsportal_hash
    ON matches_mapping
    FOR EACH ROW
    EXECUTE FUNCTION check_unique_hash_before_insert();
```

## 🎓 经验教训

### 1. 网络请求稳定性

**教训**: Playwright 页面加载可能失败或返回 404

**问题**: 26/27 个英超日期页面访问失败

**解决**:
- 增加重试机制
- 添加更长的等待时间
- 使用 wait_for_selector 等待特定元素

### 2. 队名匹配复杂性

**教训**: URL 中的队名格式与数据库中的格式不一致

**问题**: `manchester-united` vs `Manchester United`

**解决**:
- 使用 TeamNameNormalizer 进行模糊匹配
- 添加词根匹配（去除空格和连字符后比较）
- 扩展别名表覆盖更多变体

### 3. JavaScript 渲染

**教训**: OddsPortal 页面可能依赖 JavaScript 渲染

**问题**: 静态 HTML 可能不包含比赛链接

**解决**:
- 增加 Playwright 等待时间
- 使用 wait_for_selector 等待特定元素
- 考虑使用 execute_script 直接执行 JavaScript
