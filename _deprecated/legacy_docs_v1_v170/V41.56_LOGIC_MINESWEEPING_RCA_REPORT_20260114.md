### A.2 发生场景

1. **V40.22 DOM 收割器** 从 OddsPortal 提取比赛 URL:
   ```python
   # 提取 URL 末尾的 8 位哈希
   hash_code = match.get('hash')  # 例如: "tKytNkCg"
   url = "https://www.oddsportal.com/.../real-madrid-ath-bilbao-tKytNkCg/"
   ```

2. **V40.24 导入脚本** 将哈希转换为 fotmob_id:
   ```python
   fotmob_id = f"OP_{hash_code}"  # "OP_tKytNkCg"
   ```

3. **问题**:
   - 这些记录只有 oddsportal_url，**没有完整的 FotMob L2 数据**
   - fotmob_id 使用 "OP_" 前缀作为临时标识
   - 这些记录**从未被映射到真实的 FotMob match_id**
   - 导入后，V40 脚本被移到 `archive/` 目录，**清理逻辑从未执行**

### A.3 为什么只进入 matches_mapping 而不是 matches 表

**UPSERT 逻辑** (v40_24_import_fixed.py:126-150):
```sql
INSERT INTO matches_mapping (
    fotmob_id, league_name, season, home_team, away_team,
    oddsportal_hash, oddsportal_url, ...
) VALUES (...)
```

**关键发现**:
- V40.24 **只写入 matches_mapping 表**
- **从未写入 matches 表**（需要 FotMob L2 数据）
- matches 表需要真实的 FotMob external_id（纯数字）
- "OP_" 前缀的 ID **不是有效的 FotMob ID**

### B.3 根本原因

**缺陷逻辑流程**:
```
1. 系统设定服务 season = "2023/2024"
2. 扫描 OddsPortal 2023/2024 页面
3. 发现比赛 URL (包含队名 + 8位哈希)
4. 在 matches 表中搜索相同队名的比赛
5. 找到匹配（但可能是不同年份的同对阵）
6. ❌ 缺少关键检查: URL 赛季 vs 数据库赛季
7. 直接 UPSERT，导致错位
```

**缺失的校验**:
- ❌ 没有提取 URL 中的赛季字符串
- ❌ 没有对比 URL 赛季与 matches.season
- ❌ 没有验证 match_date 是否在合理范围内
- ❌ 没有拦截 OP_ 前缀的临时 ID

### B.4 影响

**抽样检查** (V41.55 随机 50 场):
- ✅ 一致: 47 场 (94%)
- ❌ 不一致: **3 场 (6%)**
- ⚠️ URL 无赛季信息: 2 场 (4%)

**外推估算**:
- 如果 6% 错误率应用到全部 1,627 个有哈希的记录
- 预估可能有 **~100 处错位**

### C.2 修复 2: 赛季错位拦截 (V41.56)

**新增代码** (第 594-622 行):
```python
# V41.56: 验证 URL 赛季与数据库赛季一致
url_season = None
if url and "[0-9]{4}-[0-9]{4}" in url:
    import re
    season_match = re.search(r'([0-9]{4}-[0-9]{4})', url)
    if season_match:
        url_season = season_match.group(1).replace("-", "/")
        # 从 matches 表获取实际赛季
        check_season_query = """
            SELECT season FROM matches WHERE match_id = %s
        """
        with self.conn.cursor() as cur:
            cur.execute(check_season_query, (match_id,))
            result = cur.fetchone()
            if result:
                db_season = result[0]
                # 检查赛季是否一致
                if url_season != db_season:
                    logger.error(
                        "🚨 V41.56: 赛季错位拦截 - match_id=%s, db_season=%s, url_season=%s",
                        match_id, db_season, url_season
                    )
                    self.stats.skipped += 1
                    return False
```

**效果**: 拦截所有 URL 赛季与数据库赛季不一致的哈希对齐。

### C.3 修复 3: 使用 matches 表的实际 season

-- =====================================================
-- V41.56: 批量检查所有潜在错位记录
-- =====================================================

-- 查找所有 URL 赛季与数据库赛季不一致的记录
SELECT
    mm.fotmob_id,
    mm.league_name,
    mm.season as db_season,
    SUBSTRING(mm.oddsportal_url FROM '[0-9]{4}-[0-9]{4}') as url_season_raw,

    m.match_date

JOIN matches m ON mm.fotmob_id = m.match_id::text
WHERE mm.oddsportal_url ~ '[0-9]{4}-[0-9]{4}'
  AND REPLACE(SUBSTRING(mm.oddsportal_url FROM '[0-9]{4}-[0-9]{4}'), '-', '/') != mm.season
ORDER BY mm.league_name, mm.season;
