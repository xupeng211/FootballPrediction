## 1. 核心引擎能力 (Engines)

### 1.1 QuantHarvester V170.000 - 数据收割机

**这是干什么用的？**
> 从 OddsPortal 网站抓取足球比赛的赔率变化轨迹数据。

**核心能力：**

| 功能 | 状态 | 说明 |
|------|------|------|
| **双模提取** | ✅ 已实现 | 网络拦截（快）→ 20秒超时后 DOM 抓取（稳） |
| **NetworkShield 集成** | ✅ 已实现 | 22节点代理池，自动轮换 |
| **Session 绑定** | ✅ 已实现 | 一场比赛 = 一个 IP，避免被识别 |
| **随机延迟** | ✅ 已实现 | 2-5秒随机间隔，模拟人类行为 |
| **自动重试** | ✅ 已实现 | 失败后自动重试，最多3次 |

**支持的数据源：**

| 数据源 | 类型 | 状态 |
|--------|------|------|
| OddsPortal | 赔率轨迹 | ✅ 主要 |
| FotMob | 比赛基础信息 | ✅ 辅助 |

**启动命令：**
```bash
# 在容器中运行
make dev-harvest

# 或直接运行
node src/infrastructure/engines/QuantHarvester.js
```

### 1.2 L1 动态发现引擎 V1.0.0 - 比赛发现

**这是干什么用的？**
> 自动发现某个联赛有哪些比赛正在进行，获取比赛 ID 列表。

**核心能力：**

| 功能 | 状态 | 说明 |
|------|------|------|
| **模式 A: API 探测** | ✅ 已实现 | 高速模式，直接调用 API |
| **模式 B: DOM 抓取** | ✅ 已实现 | API 失败时自动切换浏览器模式 |
| **跨平台映射** | ✅ 已实现 | OddsPortal → FotMob ID 映射 |
| **参数自适应** | ✅ 已实现 | 自动提取 buildId, seasonInternalId |
| **配置自愈** | ✅ 已实现 | 404 频率过高时自动更新 API 模板 |

**工作流程：**
```
[尝试 API 探测]
    ↓ 失败/404
[切换 DOM 抓取]
    ↓ 提取比赛数据
[跨平台映射]
    ↓ 查找 FotMob ID
[输出 Match ID 列表]
```

## 2. 网络与安全 (Infrastructure)

### 2.1 NetworkShield V1.1.0 - 代理管理

**这是干什么用的？**
> 统一管理所有网络请求的代理出口，防止被网站封 IP。

**核心能力：**

| 功能 | 状态 | 说明 |
|------|------|------|
| **22节点代理池** | ✅ 已实现 | 端口 7891-7912，对应 Clash Verge 节点 |
| **Session 绑定** | ✅ 已实现 | 同一个会话始终用同一个 IP |
| **智能轮换** | ✅ 已实现 | 自动选择延迟最低的节点 |
| **自愈熔断** | ✅ 已实现 | 连续失败2次后屏蔽节点，15分钟后恢复 |
| **跨语言同步** | ✅ 已实现 | Python 和 Node.js 共享代理状态 |

**熔断器状态机：**

```
[active] ──连续失败2次──→ [circuited] ──15分钟冷却──→ [尝试恢复]
    ↑                                                      │
    └────────────────────────── 成功 ──────────────────────┘
```

**配置位置：**
- `config/active_registry.json` - 节点状态持久化
