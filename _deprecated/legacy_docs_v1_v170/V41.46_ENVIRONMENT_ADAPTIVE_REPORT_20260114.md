### 1. 环境变量红线兼容 (P0)

**文件**: `src/config_unified.py`

**问题分析**:

V39.3 实施了严格的 DB_NAME 校验，只允许 `football_db`，导致本地开发环境 `football_prediction_dev` 无法启动：

```python
# V39.3 旧逻辑（过于严格）
if db_name != 'football_db':
    raise DatabaseConfigurationError(
        "🚨 非法数据库配置！"
        f"系统只允许连接 'football_db'，检测到: '{db_name}'"
    )
```

**解决方案 - V41.46 环境自适应**:

创建白名单机制，支持多环境运行：

```python
# V41.46: 环境自适应 - 智能数据库名称校验
ALLOWED_DB_NAMES = {
    'football_db',           # 生产/Docker 环境
    'football_prediction_dev', # 本地开发环境
}

# 如果设置了 DB_NAME，检查是否在白名单中
if raw_env_db_name and raw_env_db_name not in ALLOWED_DB_NAMES:
    # V41.46: 降级警告 - 允许运行但发出警告
    import warnings
    warning_msg = (
        f"⚠️  V41.46 数据库名称不在推荐列表中\n"
        f"   当前: DB_NAME='{raw_env_db_name}'\n"
        f"   推荐使用: {', '.join(sorted(ALLOWED_DB_NAMES))}\n"
        f"   系统将继续运行，但可能产生意外行为"
    )
    warnings.warn(warning_msg, UserWarning, stacklevel=2)
```

1. **`__init__` 方法 V39.3 校验** (lines 188-200)
2. **`__init__` 方法 V36.6 校验** (lines 208-216)

#### 测试 1: 5 个 Worker 分散分配

```python
def test_five_workers_get_distributed_ports(self, service):
    """
    测试: 模拟 5 个 Worker 同时调用 set_proxy_port_with_retry

    场景:
    - 5 个线程同时请求代理端口
    - 所有端口都健康（7890-7899）
    预期: 端口分配尽可能分散，避免单端口拥堵
    """
    with patch.object(service, 'check_proxy_port_health', return_value=True):
        results = []
        threads = []

        def worker():
            port = service.set_proxy_port_with_retry()
            results.append(port)

        # 启动 5 个并发 Worker
        for _ in range(5):
            t = threading.Thread(target=worker)
            threads.append(t)
            t.start()

        # 等待所有线程完成
        for t in threads:
            t.join()

        # 验证结果
        assert len(results) == 5, "应该返回 5 个端口"

        # 统计端口使用频率
        port_counts = Counter(results)

        # 验证：没有单个端口被使用超过 3 次
        for port, count in port_counts.items():
            assert count <= 3, f"端口 {port} 被使用 {count} 次，可能存在拥堵"

        # 验证：至少使用了 3 个不同的端口
        unique_ports = len(set(results))
        assert unique_ports >= 3, f"只使用了 {unique_ports} 个不同端口，分配不够分散"
```

**验证逻辑**:
- 5 个 Worker 并发请求代理端口
- 验证至少使用 3 个不同端口
- 验证单端口使用次数 ≤ 3

#### 测试 2: Worker 们尊重失效端口

```python
def test_workers_respect_excluded_ports(self, service):
    """
    测试: Worker 们尊重彼此排除的失效端口

    场景:
    - 3 个端口（7890-7892）被标记为失效
    - 5 个 Worker 并发请求端口
    预期: 所有 Worker 都避开失效端口
    """
    excluded_ports = {7890, 7891, 7892}

    def health_check(port):
        return port not in excluded_ports

    with patch.object(service, 'check_proxy_port_health', side_effect=health_check):
        results = []
        threads = []

        def worker():
            port = service.set_proxy_port_with_retry(excluded_ports=excluded_ports)
            results.append(port)

        for _ in range(5):
            t = threading.Thread(target=worker)
            threads.append(t)
            t.start()

        for t in threads:
            t.join()

        # 验证：所有端口都不在排除列表中
        for port in results:
            assert port not in excluded_ports, f"端口 {port} 在排除列表中，但仍然被使用"
```

**验证逻辑**:
- 5 个 Worker 共享 excluded_ports
- 验证所有 Worker 都避开了失效端口

#### 测试 3: 并发无阻塞

```python
def test_concurrent_performance_no_blocking(self, service):
    """
    测试: 并发调用不应该相互阻塞

    场景:
    - 5 个 Worker 同时请求端口
    - 所有端口都健康
    预期: 总耗时应该接近单次调用的耗时（无串行化）
    """
    with patch.object(service, 'check_proxy_port_health', return_value=True):
        start_time = time.time()
        results = []
        threads = []

        def worker():
            port = service.set_proxy_port_with_retry()
            results.append(port)

        for _ in range(5):
            t = threading.Thread(target=worker)
            threads.append(t)
            t.start()

        for t in threads:
            t.join()

        elapsed = time.time() - start_time

        # 验证：5 个并发调用应该在合理时间内完成
        assert elapsed < 1.0, f"并发调用耗时 {elapsed:.2f}s 过长，可能存在阻塞"

        # 验证：所有调用都成功
        assert len(results) == 5
```

**验证逻辑**:
- 5 个并发调用应在 < 1 秒内完成
- 验证无串行化阻塞
