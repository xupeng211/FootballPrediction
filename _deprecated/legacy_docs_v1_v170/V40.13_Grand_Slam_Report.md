## 🔍 技术分析

### 熔断器行为分析

**初始配置 (V40.13 第 1 版)**:
- 熔断阈值: 3 次无进展
- `slow_mo`: 0 (无延迟)
- 结果: La Liga 105 场 (触发熔断)

**优化配置 (V40.13 第 2 版)**:
- 熔断阈值: 5 次无进展
- `slow_mo`: 1000ms
- 结果: La Liga 200 场 (触发熔断)

**采集日志分析**:
```
La Liga 采集过程:
  第 1 页: 50 URL (初始)
  第 2 页: +50 URL (总计 100) ✅ 标准模式
  第 3 页: +50 URL (总计 150) ✅ 标准模式
  第 4 页: +50 URL (总计 200) ✅ 标准模式
  第 5 页: +0 URL (总计 200) ⚠️ 暴力模式，重复
  第 6-9 页: +0 URL (总计 200) ⚠️ 熔断触发
```

**问题诊断**:
- 页面 1-4 (标准模式): ✅ 正常工作，每页 50 新 URL
- 页面 5-9 (暴力模式): ❌ 返回重复内容

**根本原因**:
OddsPortal 的 `#/page/{n}/` URL 注入机制在 n ≥ 8 时可能返回重复内容，或者页面 8+ 实际不存在。

### 建议

1. **短期方案**:
   - 尝试不同的页码范围 (例如: 1-10, 11-20, 21-30)
   - 使用不同的 URL 模式 (例如: `#/page/0` vs `#/page/1`)
   - 调整暴力模式启动页 (例如: 从第 5 页开始)

2. **长期方案**:
   - 研究 OddsPortal 的 AJAX API
   - 实现基于 API 的数据采集
   - 添加队名搜索功能 (hash hunting)
