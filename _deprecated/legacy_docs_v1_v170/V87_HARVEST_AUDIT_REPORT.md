## ⚠️ 报错模式 Top 3

### 问题 1: 开盘赔率捕获率过低 (10.81%)
**影响**: 63/583 场比赛捕获到开盘赔率
**原因**: L2 hover extraction (FotMob Web) 依赖浏览器悬停交互，容易失败
**建议**:
- 优化 hover 重试机制
- 增加悬停等待时间
- 考虑替代数据源

### 问题 2: 数据完整性两极分化
**分布**:
- 完美区间 (1.02-1.08): 28.46% ✅
- 异常区间 (> 1.30): 28.08% ❌
- 过低区间 (< 1.02): 26.15% ❌

**原因**:
- 异常值可能来自数据解析错误或赔率异常波动
- 过低值可能代表赔率不完整或计算错误

**建议**:
- 添加数据清洗逻辑过滤异常值
- 检查 integrity_score 计算公式
- 分析异常样本的共同特征

### 问题 3: 联赛覆盖不均衡
**现象**:
- Serie A: 291 场 (50%)
- La Liga: 108 场 (19%)
- 其他 3 联赛: <100 场

**原因**: 收割策略可能优先处理 Serie A
**建议**:
- 并行处理多个联赛
- 提升其他联赛的收割优先级

## 📁 附录：SQL 审计脚本

### 总体资产统计
```sql
SELECT
    COUNT(*) as total_records,
    COUNT(DISTINCT match_id) as unique_matches,
    COUNT(CASE WHEN integrity_score BETWEEN 1.02 AND 1.08 THEN 1 END) as high_quality_samples,
    COUNT(CASE WHEN integrity_score > 0.98 THEN 1 END) as excellent_samples
FROM metrics_multi_source_data
WHERE source_name = 'Entity_P';
```

### 联赛分组统计
```sql
SELECT
    m.league_name,
    COUNT(DISTINCT m.match_id) as matches_harvested,
    COUNT(CASE WHEN msd.integrity_score BETWEEN 1.02 AND 1.08 THEN 1 END) as high_quality_count,
    ROUND(AVG(msd.integrity_score), 4) as avg_integrity_score
FROM metrics_multi_source_data msd
JOIN matches m ON msd.match_id = m.match_id
WHERE msd.source_name = 'Entity_P'
GROUP BY m.league_name
ORDER BY matches_harvested DESC;
```
