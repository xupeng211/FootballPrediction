```python

    "manchester": "manchester united",  # ❌ 危险！
}

# After (CORRECT)
TEAM_NAME_MAPPINGS = {
    # V41.30 FIX: REMOVED "manchester": "manchester united"
    # This prevents "Manchester City" from being incorrectly mapped
}
```

#### 2. 后缀剥离策略调整
**问题**: 剥离 "united" 和 "city" 后缀导致不同球队被混淆
```python
# Before (WRONG)
SUFFIXES_TO_STRIP = [
    r"\s+united$", r"\s+city$",  # ❌ 危险！
]

# After (CORRECT)
SUFFIXES_TO_STRIP = [
    # V41.30: REMOVED r"\s+united$" and r"\s+city$"
    # Only strip generic club suffixes like FC, AC, etc.
]
```

#### 3. 特殊字符处理增强
```python
# V41.30: Remove dots (periods)
normalized = re.sub(r"\.", "", normalized)  # "Man. Utd" → "Man Utd"

# V41.30: Remove "&" and "and"
normalized = re.sub(r"\s*[&]\s*", " ", normalized)
normalized = re.sub(r"\s+and\s+", " ", normalized, flags=re.IGNORECASE)

# V41.30: Remove hyphens
normalized = re.sub(r"-", " ", normalized)  # "West-Ham" → "West Ham"
```

## 🎓 经验教训

### 1. 数据结构理解的重要性
**教训**: 在开始工作前，必须完全理解数据结构

**问题**:
- matches 表有 380 场英超比赛
- matches_mapping 表只有 135 条记录
- 但孤儿救援脚本只发现了 43 场孤儿比赛

**原因**: JOIN 条件太严格，导致部分孤儿比赛未被识别

**解决**: 使用 NOT EXISTS 查询代替 LEFT JOIN

### 2. SQL 约束理解
**教训**: ON CONFLICT 必须匹配表的唯一约束

**问题**: `ON CONFLICT (fotmob_id, league_name, season)` 失败

**原因**: 表只有 `fotmob_id` 的唯一约束，没有组合约束

**解决**: 改为 `ON CONFLICT (fotmob_id)`

### 3. TDD 驱动开发的价值
**教训**: TDD 测试能及早发现系统漏洞

**成果**:
- 发现了 Manchester United vs Manchester City 的错误匹配
- 验证了 V41.29 青年队隔离的有效性
- 确保了核心功能的正确性
